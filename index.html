01 — The Problem

Six portals. Zero cross-referencing.

India has a wealth of publicly available MSME-relevant data — but it's scattered across entirely separate government systems with no shared taxonomy, no usability ratings, and no way to know what's machine-readable vs. a locked PDF from 2018. For a researcher at People+AI, this means every project starts with an unpaid, manual hunt before any actual analysis begins.

01
No unified index
data.gov.in, INDIAai, MOSPI, RBI, SIDBI, and Udyam all hold MSME data — but none link to each other.
02
Unknown usability
Is it a CSV or a scanned PDF? Is the data from 2024 or 2016? There's no quick answer without downloading and checking manually.
03
No coverage map
Which sectors are represented? Which geographies? Which datasets overlap? No researcher currently knows without spending days mapping it.

03 — The Build

Python scraper + Gemini summarizer

The indexing pipeline has three stages: crawl the portals to discover datasets, extract metadata (format, date, description), then pass each description through Gemini to generate a standardized usability summary. Output writes directly to a Google Sheet via the Sheets API.

# Stage 1: Discover datasets on data.gov.in import requests, pandas as pd import google.generativeai as genai BASE_URL = "https://data.gov.in/backend/dmspublic/v1/resources" SEARCH_TERMS = ["MSME", "small enterprise", "micro enterprise", "Udyam"] def fetch_datasets(query): params = { "filters[title]": query, "limit": 50, "format": "json" } resp = requests.get(BASE_URL, params=params) return resp.json().get("records", []) # Stage 2: Score usability with Gemini genai.configure(api_key="YOUR_API_KEY") model = genai.GenerativeModel("gemini-1.5-flash") SCORING_PROMPT = """ You are evaluating an Indian government dataset for MSME research usability. Dataset title: {title} Description: {desc} Format: {fmt} Last updated: {date} Return JSON with keys: usability_score (1-5), machine_readable (bool), sector_coverage (list), geographic_level (district/state/national), key_variables (list of 3-5), research_use_cases (2 sentences) """ def score_dataset(dataset): prompt = SCORING_PROMPT.format( title=dataset["title"], desc=dataset.get("description", ""), fmt=dataset.get("format", "unknown"), date=dataset.get("updated", "unknown") ) response = model.generate_content(prompt) return json.loads(response.text) # Stage 3: Write to Google Sheet def write_to_sheet(records, sheet_id): df = pd.DataFrame(records) # Uses gspread under the hood — see full code on GitHub worksheet.update([df.columns.tolist()] + df.values.tolist()) print(f"✓ Written {len(records)} datasets to sheet")

06 — Market Sizing

TAM / SAM / SOM — the MSME data opportunity

Understanding the scale of the problem anchors which data gaps matter most and where People+AI's research can have real leverage.

TAM
₹1,100 Cr+
Total Addressable Market 6.3 crore MSMEs in India contribute ~30% of GDP and 45% of exports. The full market for data-driven services targeting these enterprises — credit, compliance, market access, analytics — is estimated at over ₹1,100 crore annually.
Source: MSME Ministry Annual Report 2023; SIDBI MSME Pulse Q3 2024
SAM
~₹280 Cr
Serviceable Addressable Market ~1.5 crore digitally registered MSMEs (Udyam + GST filers) are reachable via data-driven channels. At a conservative ₹180/enterprise/year for data intelligence services, this segment alone represents ~₹280 crore.
Source: Udyam dashboard 2024; GST Council MSME filing data
SOM
Research-first beachhead
Serviceable Obtainable Market People+AI's near-term leverage isn't monetisation — it's becoming the authoritative research layer. A unified MSME data index positions the org to influence 10–15 policy and partnership engagements per year, each with direct reach to lenders, platforms, and government programs touching lakhs of enterprises.
Estimated based on People+AI existing partner network
Key data gap this index directly addresses

The single biggest barrier to unlocking the SAM is data fragmentation — lenders, policy researchers, and market platforms can't build reliable MSME intelligence without first solving discovery. The MSME Data Index is the missing infrastructure layer that makes the SAM accessible.
